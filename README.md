<p><strong>1. What is the difference between commit and push?</strong></p>
<ul align="justify">
  <li>
    Commit saves changes to your local repository as a tracked snapshot with a description.
    Push sends those local commits to a remote repository such as Github to share or back up work.
  </li>
</ul>

<p><strong>3. What causes merge conflicts?</strong></p>
<ul align="justify">
  <li>
    Merge conflicts occur when the same part of a file (or the file itself) is modified differently
    in two branches being merged, and Git cannot automatically decide which change to keep.
    This includes conflicting edits on the same lines, mismatched file additions or deletions,
    or clashing formatting changes.
  </li>
</ul>
